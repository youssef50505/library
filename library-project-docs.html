الـ Entities — نماذج قاعدة البيانات JPA Entities

الـ Entities هي كلاسات Java بيمثّل كل منها جدولاً في قاعدة البيانات. بنستخدم JPA annotations عشان نربطها بالجداول، وLombok عشان نوفر كتابة الـ boilerplate code.

Author.java 
@Entity
@Table(name = "authors")
@Data          // Lombok: getters + setters
@NoArgsConstructor // Lombok: constructor فارغ
@AllArgsConstructor// Lombok: constructor بكل الـ fields
public class Author {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @Column(nullable = false)
    private String name;  // إلزامي

    private String bio;    // اختياري

    // علاقة One-to-Many: مؤلف واحد ← كتب كتير
    @OneToMany(
        mappedBy = "author",
        cascade = CascadeType.ALL,
        fetch = FetchType.LAZY // بيجيب الكتب بس لما تطلبهم
    )
    private List<Book> books;
}
Book.java 
@Entity
@Table(name = "books")
@Data
@NoArgsConstructor
@AllArgsConstructor
public class Book {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @Column(nullable = false)
    private String title;

    @Column(nullable = false, unique = true)
    private String isbn;  // فريد — مش ينتكرر

    private int publicationYear;

    // علاقة Many-to-One: كتب كتير ← مؤلف واحد
    @ManyToOne(fetch = FetchType.LAZY)
    @JoinColumn(name = "author_id", nullable = false)
    private Author author;
}
📌 Cascade + Lazy Loading: CascadeType.ALL معناها لو حذفنا مؤلف، هتتحذف كتبه تلقائياً. FetchType.LAZY معناها الـ books مش بتتجيب من الـ database غير لما نطلبها فعلاً — ده بيحسّن الـ performance.

الـ Service Layer — منطق الأعمال Business Logic Layer

الـ Service هي أهم طبقة في التطبيق. هنا بيتنفّذ منطق الأعمال (Business Logic). بيأخذ من الـ Controller، يتحقق من البيانات، يتعامل مع الـ Repository، ويرجع نتيجة.

AuthorService.java — CREATE + UPDATE 
@Service
public class AuthorService {

    private final AuthorRepository authorRepository;

    // Constructor Injection (أحسن من @Autowired)
    public AuthorService(AuthorRepository authorRepository) {
        this.authorRepository = authorRepository;
    }

    // ======= تحويل Entity → DTO =======
    private AuthorDTO convertToDto(Author author) {
        return new AuthorDTO(
            author.getId(),
            author.getName(),
            author.getBio()
        );
    }

    // ======= CREATE =======
    @Transactional  // لو في خطأ، بيعمل rollback تلقائي
    public AuthorDTO createAuthor(AuthorDTO authorDTO) {

        // Validation — التحقق من البيانات
        if (authorDTO.getName() == null || authorDTO.getName().isEmpty()) {
            throw new RuntimeException("Author name must not be empty");
        }

        Author author = convertToEntity(authorDTO);
        Author savedAuthor = authorRepository.save(author);
        return convertToDto(savedAuthor);
    }

    // ======= UPDATE =======
    @Transactional
    public AuthorDTO updateAuthor(Long id, AuthorDTO authorDTO) {
        return authorRepository.findById(id).map(existing -> {
            existing.setName(authorDTO.getName());
            existing.setBio(authorDTO.getBio());
            return convertToDto(authorRepository.save(existing));
        }).orElseThrow(() ->
            new RuntimeException("Author not found with id " + id)
        );
    }
}
BookService.java — ميزة خاصة: منع الـ ISBN المكرر 
@Transactional
public BookDTO createBook(BookDTO bookDTO) {

    // تحقق: هل الـ ISBN موجود مسبقاً؟
    if (bookRepository.existsByIsbn(bookDTO.getIsbn())) {
        throw new RuntimeException(
            "Book with ISBN " + bookDTO.getIsbn() + " already exists"
        );
    }

    Book book = convertToEntity(bookDTO);
    return convertToDto(bookRepository.save(book));
}

// جلب كتب مؤلف معين
@Transactional(readOnly = true)
public List<BookDTO> getBooksByAuthorId(Long authorId) {

    // تحقق إن المؤلف موجود أصلاً
    if (!authorRepository.existsById(authorId)) {
        throw new RuntimeException("Author not found");
    }

    return bookRepository.findByAuthorId(authorId)
        .stream()
        .map(this::convertToDto)
        .collect(Collectors.toList());
}
⚠️ @Transactional(readOnly = true): الـ methods اللي بس بتقرأ (مش بتكتب) لازم تستخدم readOnly = true. ده بيحسّن الـ performance لأن Hibernate مش بيتتبع التغييرات.
Section 08

الـ Controllers — نقاط الدخول REST Controllers

الـ Controller هو "الباب" اللي بيستقبل الـ HTTP requests من الـ Frontend. @RestController = @Controller + @ResponseBody (يعني بيرجع JSON تلقائياً).

AuthorController.java — كل الـ Endpoints 
@RestController
@RequestMapping("/api/authors")  // الـ base URL
public class AuthorController {

    private final AuthorService authorService;

    public AuthorController(AuthorService authorService) {
        this.authorService = authorService;
    }

    // ===== POST /api/authors — إضافة مؤلف جديد =====
    @PostMapping
    public ResponseEntity<AuthorDTO> createAuthor(@RequestBody AuthorDTO dto) {
        AuthorDTO created = authorService.createAuthor(dto);
        return ResponseEntity.status(HttpStatus.CREATED).body(created);
        // يرجع HTTP 201 Created
    }

    // ===== GET /api/authors — جلب كل المؤلفين =====
    @GetMapping
    public ResponseEntity<List<AuthorDTO>> getAllAuthors() {
        return ResponseEntity.ok(authorService.getAllAuthors());
        // يرجع HTTP 200 OK
    }

    // ===== GET /api/authors/{id} — جلب مؤلف بالـ ID =====
    @GetMapping("/{id}")
    public ResponseEntity<AuthorDTO> getAuthorById(@PathVariable Long id) {
        return authorService.getAuthorById(id)
            .map(ResponseEntity::ok)
            .orElseThrow(() -> new RuntimeException("Author not found"));
    }

    // ===== PUT /api/authors/{id} — تعديل مؤلف =====
    @PutMapping("/{id}")
    public ResponseEntity<AuthorDTO> updateAuthor(
            @PathVariable Long id, @RequestBody AuthorDTO dto) {
        return ResponseEntity.ok(authorService.updateAuthor(id, dto));
    }

    // ===== DELETE /api/authors/{id} — حذف مؤلف =====
    @DeleteMapping("/{id}")
    public ResponseEntity<Void> deleteAuthor(@PathVariable Long id) {
        authorService.deleteAuthor(id);
        return ResponseEntity.noContent().build();
        // يرجع HTTP 204 No Content
    }
}

معالجة الأخطاء — GlobalExceptionHandler Global Exception Handling

بدل ما كل Controller يعالج الأخطاء لوحده، بنستخدم @ControllerAdvice عشان يكون في مكان واحد بيعالج كل الأخطاء في التطبيق.

GlobalExceptionHandler.java 
@ControllerAdvice
public class GlobalExceptionHandler {

  // RuntimeException → 404 Not Found
  @ExceptionHandler(RuntimeException.class)
  public ResponseEntity<ErrorResponse>
    handleRuntime(RuntimeException ex) {
    return new ResponseEntity<>(
      new ErrorResponse(
        ex.getMessage(),
        HttpStatus.NOT_FOUND.value(),
        LocalDateTime.now()
      ),
      HttpStatus.NOT_FOUND
    );
  }

  // Exception → 500 Internal Server Error
  @ExceptionHandler(Exception.class)
  public ResponseEntity<ErrorResponse>
    handleGeneral(Exception ex) {
    return new ResponseEntity<>(
      new ErrorResponse(
        "Internal Server Error", 500, LocalDateTime.now()
      ),
      HttpStatus.INTERNAL_SERVER_ERROR
    );
  }
}

الـ Services — HTTP Communication Angular Services

الـ Angular Services هي classes بتتعامل مع الـ HTTP requests. بنستخدم HttpClient اللي بيرجع Observable — ده زي الـ Promise بس أقوى منه (ممكن تعمله cancel، وبيدعم streams).

author.service.ts 
@Injectable({ providedIn: 'root' })
export class AuthorService {

  private readonly API =
    'http://localhost:8080/api/authors';

  constructor(private http: HttpClient) {}

  getAll(): Observable<Author[]> {
    return this.http.get<Author[]>(this.API);
  }

  create(author: Author): Observable<Author> {
    return this.http.post<Author>(this.API, author);
  }

  update(id: number, a: Author): Observable<Author> {
    return this.http.put<Author>(`${this.API}/${id}`, a);
  }

  delete(id: number): Observable<void> {
    return this.http.delete<void>(`${this.API}/${id}`);
  }
}
toast.service.ts — إشعارات للمستخدم 
export interface Toast {
  id: number;
  message: string;
  type: 'success' | 'danger' | 'warning';
}

@Injectable({ providedIn: 'root' })
export class ToastService {
  // signal: الـ Angular 16+ reactivity
  toasts = signal<Toast[]>([]);
  private counter = 0;

  show(msg: string, type = 'success') {
    const id = ++this.counter;
    this.toasts.update(list =>
      [...list, { id, message: msg, type }]
    );
    // تختفي بعد 3.5 ثانية
    setTimeout(() => this.remove(id), 3500);
  }

  success(msg: string) { this.show(msg, 'success'); }
  error(msg: string)   { this.show(msg, 'danger');  }
}

ngOnInit() {
  // forkJoin: بيعمل الـ requests كلها مع بعض في نفس الوقت
  // وبيستنى يرجعوا كلهم عشان يكمل
  forkJoin({
    authors: this.authorService.getAll(),
    books:   this.bookService.getAll()
  }).subscribe({
    next: ({ authors, books }) => {
      this.totalAuthors  = authors.length;
      this.totalBooks    = books.length;
      // آخر 5 عناصر بالترتيب المعكوس (الأحدث أولاً)
      this.recentAuthors = authors.slice(-5).reverse();
      this.recentBooks   = books.slice(-5).reverse();
      this.loading       = false;
    },
    error: () => {
      this.loading = false;
      this.error   = true;
    }
  });
}
book-list.component.ts — Search + Sort 
applyFilter() {
  const q = this.searchTerm.trim().toLowerCase();

  // فلترة بالبحث
  let result = q
    ? this.books.filter(b =>
        b.title.toLowerCase().includes(q) ||
        b.isbn.toLowerCase().includes(q)  ||
        (b.authorName ?? '').toLowerCase().includes(q)
      )
    : [...this.books];

  // Sort حسب الـ field المختار
  result.sort((a, b) => {
    const av = String(a[this.sortField] ?? '').toLowerCase();
    const bv = String(b[this.sortField] ?? '').toLowerCase();
    return this.sortDir === 'asc'
      ? av.localeCompare(bv)
      : bv.localeCompare(av);
  });

  this.filtered = result;
}

title VARCHAR NOT NULL
isbn VARCHAR UNIQUE NOT NULL
publication_year INT
🔗 author_id BIGINT (FK)

author_id في جدول books هو الـ Foreign Key اللي بيربط الكتاب بمؤلفه

⚠️ CascadeType.ALL: لو حذفت مؤلف، هيتحذف كل كتبه تلقائياً! ده بسبب cascade = CascadeType.ALL في الـ Author entity. تأكد إنك ما تحذفش مؤلف وأنت مش عارف إن ليه كتب.

تشغيل المشروع خطوة بخطوة Project Setup Guide

🖥️ Backend Setup

  1. تثبيت المتطلبات

    لازم يكون عندك Java 21 وPostgreSQL مثبتين.

  2. إنشاء قاعدة البيانات

    افتح pgAdmin أو psql وشغّل:

    CREATE DATABASE project_full_lib;
  3. اضبط الـ application.properties

    تأكد إن الـ username والـ password صح في ملف application.properties.

  4. شغّل الـ Backend 
    // من مجلد library2
./mvnw spring-boot:run

// أو من IntelliJ: Run Library2Application
  5. تأكد إنه شغّال

    افتح المتصفح على http://localhost:8080/api/authors — المفروض ترجع []

🌐 Frontend Setup

  1. تثبيت Node.js و Angular CLI 
    npm install -g @angular/cli
  2. تثبيت الـ dependencies 
    // من مجلد library-frontend
npm install
  3. تشغيل الـ Frontend 
    ng serve

// أو
npm start
  4. افتح في المتصفح

    روح على http://localhost:4200 وهتلاقي الـ Dashboard.

⚠️ مهم: الـ Backend لازم يكون شغّال أول قبل ما تفتح الـ Frontend، وإلا الـ Dashboard هيطلع خطأ.
